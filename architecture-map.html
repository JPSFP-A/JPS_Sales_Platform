Sales Platform

Single-file SPA — kWh sales analysis, LE forecasting, revenue recovery, defection tracking. Auth-gated via Supabase SSO shared across the JPS Analytics Suite.

Notable Findings
Schema stale banner risk: PostgREST cache drift silently breaks uploads and deletes. Auto-banner detection in place but relies on a manual DB schema SQL run.
Dual analytics stack: Both Sentry (error tracking) and PostHog (product analytics) load on every page — adds ~200 KB and two external network calls before first paint.
XLSX loaded lazily: The SheetJS library is not in the initial bundle; it is injected on first upload/download — saves ~1 MB on initial load but adds latency on first use.
LE Commit writes to fpa_facts: The LE forecast commit path crosses into the FP&A Finance Hub DB tables (fpa_versions, fpa_facts, fpa_derived_formulas) — tightly coupling the Sales Platform to the Finance Hub schema.
Admin wipe-all function: doWipeAllActuals() deletes every row from jps_actuals with a .gte('id',1) — destructive with no server-side safeguard beyond a client-side confirmation code.
